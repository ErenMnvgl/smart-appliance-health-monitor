# Database Design

This document describes the main database tables of the project.

## Device

Represents a smart appliance.

Fields:

- id
- deviceCode
- name
- type
- location
- status
- lastSeenAt
- createdAt

Example device types:

- REFRIGERATOR
- WASHING_MACHINE
- AIR_CONDITIONER
- TV
- DISHWASHER

Example statuses:

- ONLINE
- OFFLINE
- WARNING
- ERROR

## SensorLog

Represents sensor data sent by a device.

Fields:

- id
- deviceId
- temperature
- energyUsage
- voltage
- signalStrength
- createdAt

## ErrorLog

Represents an error generated by a device.

Fields:

- id
- deviceId
- errorCode
- message
- severity
- resolved
- createdAt

Example severity values:

- LOW
- MEDIUM
- HIGH
- CRITICAL

## ConnectionLog

Represents online/offline history of a device.

Fields:

- id
- deviceId
- status
- reason
- createdAt
